These are the official rules for the VibeCodeUtah Hackathon. By participating, you agree to abide by all rules listed below.

## Eligibility Requirements

<Aside type="caution" title="Important">
  You must meet ALL eligibility requirements to be considered for prizes.
</Aside>

- **Open to all skill levels** — Beginners and experts alike are welcome
- **Age requirement** — Must be 18 years or older, or have parental consent
- **Team size** — Solo participants or teams of up to 4 people
- **Registration** — All team members must be registered before the hackathon starts

## Open Source Requirement

<Aside type="tip" title="MIT License Required">
  All submissions must be open source under the **MIT License** to be eligible for prizes.
</Aside>

Your code becomes a gift to the community. This requirement ensures:
- Transparency in judging
- Knowledge sharing with the developer community
- Long-term impact beyond the hackathon

### How to Add MIT License

1. Include a `LICENSE` file in your repository root
2. Use the standard MIT License text
3. Add the current year and your name/team name

```text
MIT License

Copyright (c) 2025 [Your Name/Team Name]

Permission is hereby granted, free of charge, to any person obtaining a copy...
```

## Project Requirements

### What Must Be New
- Projects must be started fresh for this hackathon
- No pre-existing projects or significant pre-written code
- Start from scratch or from a basic boilerplate/template

### What's Allowed
- Third-party libraries and packages (npm, pip, etc.)
- APIs and external services
- Starter templates and boilerplates
- AI coding assistants (required!)

### What's NOT Allowed
- Pre-existing projects with substantial functionality
- Code written before the hackathon start date
- Purchasing or copying others' code
- Violating any third-party terms of service

## Submission Requirements

<Steps>
1. **GitHub Repository**
   - Public repository with MIT License
   - Clear README with setup instructions
   - Clean commit history showing development progress

2. **Video Demos (Vibe Sessions)**
   - 3 sessions, each 20-30 minutes
   - Screen recording with audio narration
   - Show your AI tools and decision-making process

3. **Project Documentation**
   - What problem does it solve?
   - How was AI used in development?
   - Setup and installation instructions

4. **Final Demo Video**
   - 3-5 minute polished demo
   - Showcase key features
   - Explain the value proposition
</Steps>

## Judging Criteria

Projects will be evaluated on:

| Criteria | Weight | Description |
|----------|--------|-------------|
| **Innovation** | 25% | Originality and creativity of the solution |
| **Execution** | 25% | Technical implementation quality |
| **AI Utilization** | 20% | Effective use of AI tools in development |
| **Impact** | 15% | Potential real-world value |
| **Presentation** | 15% | Quality of demos and documentation |

## Code of Conduct

All participants must:
- Treat others with respect and professionalism
- Provide constructive feedback
- Report any violations to organizers
- Comply with all applicable laws

Violations may result in disqualification.

## Timeline Compliance

- All code must be committed within the official hackathon dates
- Late submissions will not be accepted
- Timestamps in git history will be verified
